Sort button — how it works
On-demand classification for selected emails.
1
Select emails
Tick the checkboxes on any emails in the Gmail inbox.
2
Click Sort
The extension reads sender, subject, and snippet from each selected row.
3
Classify
Keyword rules run first. If no match and Gemini mode is on, Gemini 2.0 Flash picks a label. Results are returned to the extension.
4
Apply labels via Apps Script
The extension POSTs { token, applications: [{ messageId, label }] } to the Apps Script web app. Apps Script uses the Gmail API to apply labels.
5
Toast confirmation
A toast shows "Labeled X email(s)". Labels appear on the emails immediately.
⚠️ Sort applies labels only — it never archives. The Layer 2 auto-sorter handles archiving on its 5-minute sweep.
🏗️
Three-layer architecture
Unknown senders fall through each layer in sequence.
1
Instant · Server-side
XML filters
69 Gmail server-side rules. Known senders are labeled the moment mail arrives — no compute, no latency. Managed in gmail-filters.xml. Re-import into Gmail when rules change.
↓
2
Every 5 minutes · Apps Script
Auto-sorter
Sweeps unlabeled inbox mail. Matches against the same 69 JS rules (in rules.gs). If no match and Gemini mode is on, calls Gemini 2.0 Flash. Applies label + archives. Logs unknown senders to the Review Queue sheet for manual triage.
↓
3
Manual · Chrome extension
Extension (this tool)
Tab bar for quick label navigation. Sort button for on-demand classification of emails you select. Guide overlay for reference.

Label guide
What each label means and when it's applied.
📰
Newsletter
Substack, blogs, digests, and recurring content emails. Archived.
🔔
Notification
App alerts, shipping updates, service messages. Archived.
🧾
Receipt
Purchases, payments, invoices, billing confirmations. Archived.
📅
Calendar
Appointment reminders, event invitations, scheduling links. Archived.
📣
Marketing
Promotions, sales, brand emails, product launches. Archived.
❄️
Cold Email
Unsolicited outreach, sales pitches from unknown senders. Archived.
🔐
Security
Password resets, 2FA codes, account alerts. Archived.
👤
Personal
Real person, personal conversation. Never archived — stays in inbox.
ℹ️
FYI
Info-only, no action needed, doesn't fit other categories. Archived.
💬
To Reply
Manual label — requires your response. Stays in inbox.
🔖
Follow-up
Manual label — you need to follow up. Stays in inbox.
✅
Actioned
Manual label — task complete. Archive when done.
💼 Job search senders (Greenhouse, Lever, Workday, ZipRecruiter, LinkedIn jobs) are whitelisted — the auto-sorter never touches them. They stay in inbox with no label.

Extension settings
Click the Gmail Forge icon in Chrome's toolbar to open the popup.
Classifier Mode
Gemini — uses AI for unknown senders (Gemini 2.0 Flash). Rules Only — keyword regex only, zero API calls. Default: Gemini.
Gemini API Keyoptional
From aistudio.google.com. Required only in Gemini mode. Stored locally in Chrome sync storage.
Web App URLrequired for Sort
The Apps Script deployment URL. Get it from script.google.com → Deploy → Manage deployments. Without this, Sort classifies but can't apply labels.
Trigger Tokenrequired for Sort
Must match the TRIGGER_TOKEN Script Property in Apps Script. Authorises label-apply requests from the extension.
Visible Tabs
Toggle which label tabs appear in the toolbar. Unchecked tabs are hidden but still work as Gmail label views.
➕
Adding new rules
When a new sender consistently needs a specific label.
1
Check the Review Queue
Open the Google Sheet → Review Queue tab. Unknown senders logged by the auto-sorter appear here. Fill in "Assign Label" for any you want to lock in.
2
Add to rules.gs
Add the domain (preferred) or exact address to the correct label in apps-script/rules.gs. Domain-level matching catches all variants from a sender.
3
Add to gmail-filters.xml
Add a matching entry in gmail-filters.xml so the server-side Layer 1 filter also catches it instantly.
4
Deploy
Run cd apps-script && npx clasp push --force to push rules.gs to Apps Script.
5
Re-import XML
Gmail → Settings → Filters → Import filters → upload gmail-filters.xml → check "Also apply to matching conversations" → Create filters.
