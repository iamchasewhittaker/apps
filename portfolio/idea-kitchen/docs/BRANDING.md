# Branding — Idea Kitchen

> See `docs/templates/PORTFOLIO_APP_BRANDING.md` for the full template this is based on. This doc is scoped to what a docs-only app actually needs.

## Name
**Idea Kitchen.** Plain English, 2 words, no invented spellings. Kitchen = where raw ingredients (ideas) become finished dishes (shipped apps). Not clever — descriptive.

## Palette

Docs-only app, so palette lives in tiny places: a potential future logo, the Shipyard tile background, and any static HTML minisite we might generate.

| Role | Hex | Notes |
|---|---|---|
| Background | `#0f1117` | Portfolio dark base (matches Knowledge Base, Wellness). |
| Surface | `#1a1d26` | One step up from base. Cards, panels. |
| Primary | `#f5a524` | Warm amber — "kitchen" cue, pairs with dark base. |
| Accent | `#e5e7eb` | Near-white for high-contrast text on dark. |
| Text | `#e5e7eb` / `#9ca3af` | Primary / muted. |

## Typography

- **Primary:** DM Sans — portfolio default. Clean, readable at body size.
- **Monospace:** system mono (`ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas`) — for the code blocks in BUILD_GUIDE.md and the prompt file.
- **Hierarchy:** H1 (28–32px), H2 (22–24px), H3 (18px), body (16px), small (14px).

## Voice

Idea Kitchen talks like a patient, experienced PM who's done this loop enough times to know the shortcuts. Short sentences. No hype. Teaches without condescending. Says *why* when a call isn't obvious.

Identity voice rules apply (no em-dashes, no rule-of-threes, no consultant phrasing, no hype words like *leverages*, *unlocks*, *compounds*, *synergy*). Warm, direct, plain.

## Logo direction

No logo in Phase 1. If Phase 2 needs one:

- **Concept:** a stylized pot or bowl with steam rising, or a simple "IK" monogram.
- **Format:** follow `docs/templates/PORTFOLIO_APP_LOGO.md` — dark background (`#0f1117`), amber accent (`#f5a524`), white main mark, DM Sans wordmark.
- **Where it goes:** Shipyard tile, favicon if we ever build a static HTML minisite.

## Do / don't

**Do:**
- Let section structure carry the weight. H2 + H3 > prose paragraphs.
- Lead with the rule or decision. Reasoning follows.
- Use fenced code blocks for any command, path, or prompt snippet.

**Don't:**
- Decorate with emoji.
- Use em-dashes. Use periods, colons, or parentheses instead.
- Use the rule of three ("fast, simple, and powerful") — it's a hype tell.
- Say *"we"* when you mean *"you"*.
