# Product Brief — Idea Kitchen

- **App summary.** A reusable system that turns a raw idea into a shipped portfolio app through one Claude Project (ideation) and one Claude Code kickoff (build).
- **Target user.** Chase — solo builder running the `~/Developer/chase/` portfolio.
- **Main pain.** Every new idea costs the same re-explanation: portfolio conventions, voice rules, security rules, scaffold commands. Drift between plan and code. No visual surface for "what am I making." Tokens wasted on answered questions.
- **Core value.** One loop: ideate → artifacts → scaffold → build → ship. Same conventions every time. Less repetition. Patterns captured so the next loop is cheaper.
- **V1 scope.**
  - Claude Project system prompt with 10-step flow that produces 6 handoff artifacts.
  - Master build guide covering setup, per-idea workflow, end-of-session ritual, security, best practices.
  - SHOWCASE template (Shipyard-parseable) + Project Knowledge manifest + filled SESSION_START example.
  - Dogfooded Phase 1–3 docs for Idea Kitchen itself.
  - Seeded `identity/patterns.md` for cross-session pattern capture.
