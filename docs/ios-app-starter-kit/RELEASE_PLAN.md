# Release Plan

Use this to prepare for beta and App Store release.

## Pre-release checklist
- confirm V1 scope is locked
- finish QA pass
- confirm app icon
- prepare screenshots
- prepare privacy details
- review onboarding or first-run copy
- confirm support/contact info if needed

### What to validate
- main flow is understandable
- no blocking bugs
- performance is acceptable
- theme does not create confusion

### Feedback questions
- What was confusing?
- What felt useful?
- What felt unnecessary?
- Would you use this again tomorrow?

## App Store prep
- app name
- subtitle
- keywords
- description
- screenshots
- privacy labels
- age rating
- support URL if needed

## Launch risks
- last-minute scope creep
- hidden crashes on device
- unclear onboarding
- unsupported edge cases

## Post-launch plan
- monitor crashes
- collect feedback
- prioritize V1.1 fixes
- create V2 roadmap based on actual usage
